# Ходилка

Веб-приложение для проведения учебной игры с тремя интерфейсами:
ученик, ведущий и проектор.
Фронтенд статический,
состояние комнат, игроков, вопросов и ответов хранится в `Supabase`.

## Что есть в проекте

- `index.html`:
  экран ученика для входа в комнату и отправки ответов.
- `admin.html`:
  панель ведущего для авторизации, создания комнат,
  загрузки вопросов и проверки ответов.
- `projector.html`:
  отдельный экран комнаты для вывода вопроса,
  поля и таблицы результатов.
- `src/`:
  клиентская логика и стили.
- `assets/`:
  фон и изображения фишек игроков.
- `supabase/`:
  SQL-схема и конфиг подключения клиента.
- `docs/`:
  инструкция по запуску, примеры и исходные материалы.

## Архитектура

Приложение использует один модуль
[script.js](/Users/nikit/Downloads/Ходилка - v1/src/script.js),
который переключает поведение по `data-app` в `body`.
За счёт этого логика комнаты,
подписки на realtime-обновления и работа с `Supabase`
собраны в одном месте,
а HTML-страницы остаются тонкими точками входа.

Визуальная часть вынесена в
[style.css](/Users/nikit/Downloads/Ходилка - v1/src/style.css).
Поле игры рисуется как фиксированный маршрут,
а позиции игроков обновляются на клиенте по данным комнаты.

## Быстрый старт

1. Настрой `Supabase` по инструкции из
   [docs/SETUP.md](/Users/nikit/Downloads/Ходилка - v1/docs/SETUP.md).
2. Заполни
   [supabase/config.js](/Users/nikit/Downloads/Ходилка - v1/supabase/config.js).
3. Подними любой статический сервер,
   например `python3 -m http.server 8080`.
4. Открой `http://localhost:8080/index.html`.

## Основные сценарии

1. Ведущий открывает `admin.html`,
   входит в систему и создаёт комнату.
2. Ученики открывают `index.html`,
   вводят код комнаты и своё имя.
3. Ведущий загружает JSON с вопросами и открывает нужный вопрос.
4. Проектор открывается через `projector.html`
   или по ссылке из админки.
5. После ответа ведущий проверяет результаты и завершает игру.
